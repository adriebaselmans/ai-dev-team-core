# Review Findings Template

Use this format for reviewer output.

Write the active review to `doc_templates/review/current.yaml`.

## Decision
- `Approved for testing` or `Rework required`

## Findings
- List findings ordered by severity.
- Include file references where possible.

## Residual Risks
- Record any remaining concerns that are not blocking.

## Recommendation
- State whether the coordinator should proceed to testing or route back to development or architecture.
